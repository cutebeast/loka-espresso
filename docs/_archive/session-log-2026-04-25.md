## What Was Accomplished

### 1. JWT Security Hardening

**Removed tokens from JSON response bodies**
- `POST /auth/verify-otp` — now returns `{"is_new_user": bool}` only; tokens in httpOnly cookies
- `POST /auth/login-password` — now returns `{"message": "Login successful"}` only
- `POST /auth/refresh` — now returns `{"message": "Token refreshed"}` only
- **Files**: `backend/app/api/v1/endpoints/common/auth.py`

**Rate-limited refresh endpoint**
- Added `@limiter.limit("30/minute")` to `/auth/refresh`

**Blacklist logging and error handling**
- Decode failures during blacklist logged at `warning` (was `debug`)
- DB errors during blacklist now propagate (was silently caught and continued)

**Token creation**
- `security.py` still uses `python-jose`; PyJWT migration started with `security_v2.py` import in `auth.py`
- Fixed `NameError` in `require_role` and `require_store_access` dependency factories (return variable mismatch)

### 2. Runtime Config Security Hardening

**Allowlist enforcement**
- `PUT /admin/config` rejects keys not in `ALLOWED_CONFIG_KEYS`
- `otp_bypass_code` removed from allowlist entirely
- `SENSITIVE_KEY_PATTERNS` expanded to include `"bypass_code"`

**Response filtering**
- `GET /admin/config` filters out sensitive keys before returning
- Audit log redacts sensitive old/new values as `***REDACTED***`

**File**: `backend/app/api/v1/endpoints/common/config.py`

**Configurable refresh TTL**
- Added `JWT_REFRESH_EXPIRE_DAYS` to `Settings` (default 30)

**OTP logging**

**Files**: `backend/app/main.py`, `backend/app/core/config.py`, `backend/app/services/sms.py`
